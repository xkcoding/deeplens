## MODIFIED Requirements

### Requirement: Index page generation
The system SHALL generate an `en/index.md` document as the VitePress home page from the outline's `overview` data. The `en/index.md` SHALL contain: project name as H1 title, `overview.architecture` rendered as a section with Mermaid diagram, `overview.tech_stack_roles` as a table (columns: Technology, Role), `overview.key_flows` as a numbered description list, `overview.project_structure` as a descriptive section, navigation links to each domain's hub document, a Quick Start section with 2-3 actionable next steps for new developers linking to specific domain docs, and a "Generated by DeepLens" attribution.

#### Scenario: Home page generated from overview
- **WHEN** the generation agent processes a confirmed outline with overview data
- **THEN** `en/index.md` contains architecture section, tech stack table, key flows, structure description, domain navigation links, and Quick Start section

#### Scenario: Home page links to domains
- **WHEN** the documentation site is opened
- **THEN** the index page shows the project name, architecture overview, and clickable links to each domain's hub document under `/en/domains/`

#### Scenario: Quick Start section
- **WHEN** the overview generator completes en/index.md
- **THEN** the Quick Start section provides 2-3 concrete, actionable steps linking to specific domain docs

### Requirement: Architecture overview document
The system SHALL NOT generate a separate top-level `architecture.md` document. All architecture content (project summary, detected technology stack, system architecture diagram, domain index table) SHALL be consolidated into `en/index.md` via the overview data. This avoids content duplication between `architecture.md` and `en/index.md`.

#### Scenario: No standalone architecture.md
- **WHEN** all domain documents are complete
- **THEN** no `architecture.md` file exists in any docs directory; architecture content is in `en/index.md`

### Requirement: Summary page generation
The system SHALL generate an `en/summary.md` document as the project summary / wrap-up page AFTER the overview is generated. The `en/summary.md` SHALL be produced by an independent `runSummaryGenerator()` sub-agent that reads the overview and all domain hub documents. The `en/summary.md` SHALL contain: "Project Summary" as H1 title, At a Glance table (key metrics), Domain Recap (1-2 sentences per domain), Cross-cutting Concerns (patterns spanning domains), Architectural Insights (global perspective), Potential Improvements (tech debt, gaps), and "Generated by DeepLens" attribution.

#### Scenario: Summary generated after overview
- **WHEN** the overview generator completes en/index.md
- **THEN** the summary generator runs and produces `en/summary.md`

#### Scenario: Summary content is evidence-based
- **WHEN** the summary generator reads domain docs
- **THEN** the Domain Recap section references actual content from domain hub documents under `en/domains/`

#### Scenario: Summary phase progress
- **WHEN** the summary generator starts
- **THEN** a progress event with `phase: "summary"` and `completed: 0, total: 1` is emitted
- **AND WHEN** en/summary.md is written
- **THEN** a progress event with `phase: "summary"` and `completed: 1, total: 1` is emitted

## ADDED Requirements

### Requirement: English document output path prefix
The Generator Agent, Overview Generator, and Summary Generator SHALL write all documents under the `en/` directory prefix. Domain documents SHALL be written to `en/domains/{domain-id}/index.md` and `en/domains/{domain-id}/{spoke}.md`. The overview SHALL be written to `en/index.md`. The summary SHALL be written to `en/summary.md`.

#### Scenario: Domain docs under en/ prefix
- **WHEN** the generator writes hub and spoke documents for domain "auth"
- **THEN** files are written to `en/domains/auth/index.md` and `en/domains/auth/{spoke}.md`

#### Scenario: Overview under en/ prefix
- **WHEN** the overview generator writes the home page
- **THEN** the file is written to `en/index.md`

#### Scenario: Summary under en/ prefix
- **WHEN** the summary generator writes the summary page
- **THEN** the file is written to `en/summary.md`
