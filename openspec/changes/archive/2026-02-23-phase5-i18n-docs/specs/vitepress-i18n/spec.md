## ADDED Requirements

### Requirement: VitePress i18n configuration
The VitePress scaffold SHALL generate a `config.mts` using VitePress native `locales` configuration with two locales: `en` (English, default) and `zh` (Chinese). Each locale SHALL have its own `label`, `lang`, `link` prefix, and `themeConfig` with independent sidebar and nav.

#### Scenario: Locales configuration generated
- **WHEN** VitePress is scaffolded
- **THEN** `config.mts` contains a `locales` object with `en` and `zh` entries

#### Scenario: English as default locale
- **WHEN** a user visits the documentation site root
- **THEN** the site serves the English version under `/en/`

#### Scenario: Language switcher in nav
- **WHEN** a user views any documentation page
- **THEN** the top navigation bar displays a language switcher with "English" and "中文" options

### Requirement: Bilingual sidebar generation
The `generateSidebar()` function SHALL accept a `locale` parameter (`"en"` or `"zh"`) and generate sidebar configuration for the specified language. The function SHALL scan the corresponding locale directory (`en/domains/` or `zh/domains/`) for actual files. The sidebar links SHALL use locale-prefixed paths (e.g., `/en/domains/auth/` or `/zh/domains/auth/`).

#### Scenario: English sidebar generation
- **WHEN** `generateSidebar(outline, docsDir, "en")` is called
- **THEN** sidebar items link to `/en/` prefixed paths: `/en/`, `/en/domains/auth/`, `/en/summary`

#### Scenario: Chinese sidebar generation
- **WHEN** `generateSidebar(outline, docsDir, "zh")` is called
- **THEN** sidebar items link to `/zh/` prefixed paths: `/zh/`, `/zh/domains/auth/`, `/zh/summary`

#### Scenario: Sidebar reflects actual files
- **WHEN** English has 5 domains but Chinese translation only completed 3
- **THEN** English sidebar shows 5 domain groups, Chinese sidebar shows only the 3 translated domain groups

### Requirement: Bilingual directory structure
The VitePress scaffold SHALL create the following directory structure:

```
.deeplens/docs/
├── en/                  # English docs (generated by Generator)
│   ├── index.md
│   ├── summary.md
│   └── domains/
├── zh/                  # Chinese docs (generated by Translator)
│   ├── index.md
│   ├── summary.md
│   └── domains/
└── .vitepress/
    └── config.mts       # i18n config with locales
```

#### Scenario: Directory scaffold with i18n
- **WHEN** VitePress is scaffolded for a project
- **THEN** both `en/` and `zh/` directories exist under the docs root

#### Scenario: Config references both locales
- **WHEN** config.mts is generated
- **THEN** the `locales` object references sidebar data for both `/en/` and `/zh/` paths
